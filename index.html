Snabb

micro PHP framework

View project on GitHub

What is this?

Is a compilation about snippets that I found over internet and put them together to work as a micro framework for PHP. At the moment its main work is for resolve routes and provide render view. Data access to DB or rest Services must be implemented.

How to use it?

First you download, fork or clone the repository. And add a couple of files to have your site running.

Here is how you can use it:

Routes

This modules is from a github repo (php-rails-routing). I made some modifications. In order to resolve controller Class, action (instance method) and view.

Defining routes

Go to folder config and open routes.php

and add your routes as follow:

  <?php

  // mapping for http://<location>/hello
  $r->map("hello", "hello::world");

  ?>

where:

  • "hello" is a route on your site
  • "hello::world" is a mixed term controllerName::actionName

so router will runs

// -> HelloController->worldAction()

Defining controllers

In the app/controllers folder you put the controllers to handle your routes

the convention name is <ControllerName>Controller.php

controllers are clases like follow:

<?php

  class HelloController {

    //Action related to route
    public function worldAction() {
      //We just need to return the data to be render at view (html, json or xml)
      return [ "title" => "Hello World!" ] ;
    }

}
?>

Defining Views

In the app/views folder you put the views (html) to handle render of your routes

Yu will need to create a folder by each controller you have defined on app/controllers folder.

the convention name is /<controllername>/<actionname>.tpl.php

views are files like follow:

view file path: app/views/hello/world.tpl.php

  <p><?= $title ?></p>
